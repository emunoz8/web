# JIT Cafe
### React Restaurant Website
**03/11/26**

JIT Cafe is a guest-facing restaurant website built as a React project. It is designed like the public website for a restaurant, with a home page that introduces the brand, a food menu page that acts as the main experience, and a contact page that gives the site a complete business-facing surface.

The main purpose of the website is to show how a restaurant menu can carry the identity of the site instead of feeling like a utility page. The layout, card design, category browsing, typography, spacing, and imagery are all meant to make the menu feel readable, polished, and central to the overall experience.

On this site, the project is presented as a frontend feature route focused on the public restaurant surface. The menu is driven by static snapshot data, the layout is responsive, and the interface is meant to feel more like a finished restaurant website than a generic app shell.

## What the website is

At its core, this is a restaurant website prototype centered on the customer-facing experience. Instead of focusing on dashboards or admin tools, it focuses on what a visitor would actually see first: the restaurant identity, the menu, and the contact information.

The site is intentionally simple in structure, but that simplicity is part of the point. Each page has a clear job:

- the home page introduces the restaurant and sets the visual tone
- the menu page does the main browsing work and acts as the centerpiece
- the contact page rounds out the site with the basic business-facing information

## What I like most about this project

What I like most is that the menu actually feels like the product. A lot of restaurant sites treat the menu as an afterthought, but here the menu is the main visual and structural focus of the experience. That made the project more interesting to me because it was less about dropping content onto a page and more about shaping the browsing experience itself.

I also like that the project let me combine UI structure, visual design, and practical frontend architecture in one piece of work. It gave me a chance to build something that feels complete as a public-facing website while still fitting cleanly into the larger site as a React feature.
